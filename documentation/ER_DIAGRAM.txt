RELATIONSHIP SUMMARY
---------------------
- Users (1) ----< (N) HealthRecords        : one user can have many health records
- Users (1) ----< (N) Predictions          : one user can have many predictions
- Users (1) ----< (N) Feedback             : one user can submit many feedback entries
- Users (1) ----< (N) Reports              : one user can generate many reports
- HealthRecords (1) ----< (N) Predictions  : one health record can produce predictions
- Models (1) ----< (N) Predictions         : one model can be used for many predictions
- Predictions (1) ----< (N) Reports        : one prediction can have multiple reports generated from it

All foreign keys enforce referential integrity via SQLite's
"PRAGMA foreign_keys = ON" setting (see database/db_manager.py).
